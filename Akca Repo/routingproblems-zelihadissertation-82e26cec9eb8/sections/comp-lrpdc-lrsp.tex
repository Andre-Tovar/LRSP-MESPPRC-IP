\section{Effect of Scheduling Constraint in the LRSP}

In this section, we experimentely show the
benefit of optimizing scheduling decision integrated with
the location and routing decisions. In other words, the gain
that can be obtained by solving the LRSP. To be able to measure
this benefit, we solve the LRP with distance constraints (LRPDC). 
In LRPDC, each route corresponds to a vehicle. As the vehicle
capacity gets larger, we expect that the number 
of vehicles used in optimal solution of the LRSP gets closer to the
number of vehicles used in the LRPDC. Therefore, solving the LRSP
gets closer to  solving the LRPDC. 
%However, it may be
%possible to decrease the number of vehicles used by assigning multiple
%routes to a single vehicle, in other words solving scheduling problem
%for the generated routes. 
After solving the LRPDC, bin packing problem 
is solved to assign routes to vehicles considering the time limit. 
Therefore, scheduling decision has been made independent of the location and 
routing decisions. 

The branch and price algorithm for the LRSP can easily be changed to 
solve the LRPDC. The master problem formulations for both LRSP and LRPDC
are same. These two problems differ in the definitions of their columns.   
A column in LRSP represents a set of routes while a column in 
LRPDC represent a single route feasible with respect to vehicle capacity and
time limit. Therefore, the pricing problem for the LRPDC can be formulated
as an ESPPRC where the vehicle capacity and time limit are the resources.
In the 2 phase pricing problem designed for the LRSP, the first phase 
generates routes using ESPPRC with vehicle capacity and time limit resources.
And the second phase combines these routes into pairings using ESPPRC
with time limit as a resource. If we only solve the 
first phase, the problem generates feasible columns for the LRPDC. 
Therefore, by changing the pricing problem in the branch and price algorithm,
we can solve the LRPDC. 

{\bf Instances} 
We used the lrsp instances generated from \citet{Cordeau} 
with 25 customers. We considered instances with the first time limit value (140).    
For each location set, we used 4 different vehicle capacity values. 
In addition to $C^V_1$, $C^V_2$ values, we generated 2 larger vehicle 
capacities refered as $C^V_3$ (generated by setting $\alpha = 0.5$) and $C^V_4$ 
(generated by setting $\alpha = 0.8$).
